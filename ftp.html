O que é FTP?

FTPS (FTP Secure ou FTP-SSL) é uma extensão do protocolo FTP que adiciona suporte para criptografia usando o SSL (Secure Sockets Layer) ou o seu sucessor, o TLS (Transport Layer Security). O objetivo do FTPS é proteger a transmissão de dados e as informações de autenticação contra interceptação e ataques.

Para que serve?

FTPS serve para:

  • Segurança de Dados: Protege a transferência de arquivos através de criptografia, o que é especialmente importante para informações sensíveis.
  • Autenticação Segura: Garante que as credenciais (nome de usuário e senha) sejam transmitidas de forma segura.
  • Conformidade: Atende a requisitos de segurança e conformidade que exigem criptografia para a transferência de dados.

Como funciona?

  • Estabelecimento de Conexão: O cliente FTPS se conecta ao servidor FTPS da mesma forma que o FTP tradicional, mas utilizando o SSL/TLS para estabelecer uma conexão segura.
  • Negociação de Segurança: O cliente e o servidor negociam as opções de segurança, como os protocolos SSL ou TLS e os métodos de criptografia que serão usados.
  • Autenticação: Durante o processo de autenticação, as credenciais são criptografadas, tornando a captura de informações confidenciais muito mais difícil.
  • Transmissão de Dados: Tanto a conexão de controle (para comandos e respostas) quanto a conexão de dados (para a transferência real dos arquivos) são protegidas pela criptografia. Dependendo da configuração, o FTPS pode usar criptografia explícita (onde a conexão começa não criptografada e é depois convertida para segura) ou implícita (onde a conexão é estabelecida diretamente com criptografia).

Tipos de FTPS:


FTPS Explícito:

  • USER e PASS: Autenticação do usuário.
  • LIST ou NLST: Listar arquivos e diretórios no servidor.
  • RETR: Baixar um arquivo do servidor.
  • STOR: Enviar um arquivo para o servidor.
  • DELE: Deletar um arquivo no servidor.
  • MKD e RMD: Criar e remover diretórios, respectivamente.
  • QUIT: Encerrar a conexão FTP.

Como foi feito?

Desenvolvimento Inicial:

  • O FTP foi desenvolvido na década de 1970 como um dos primeiros protocolos para a transferência de arquivos entre sistemas em rede.
  • A primeira especificação foi publicada em 1971, e a versão mais recente do protocolo foi formalizada em 1985.

Evolução:

  • O FTP foi inicialmente definido para ser simples e eficaz em redes locais. Com o crescimento da internet, as implementações do FTP foram adaptadas para lidar com maior segurança e eficiência.
  • Novas versões e extensões foram desenvolvidas para melhorar a funcionalidade, como FTPS (FTP Secure) e SFTP (SSH File Transfer Protocol), que adicionam segurança adicional ao processo de transferência.

Variações e Extensões

FTPS (FTP Secure):

  • Segurança: Adiciona uma camada de segurança usando SSL/TLS para criptografar a comunicação entre o cliente e o servidor FTP.
  • Portas: Utiliza as mesmas portas do FTP padrão, mas pode ser configurado para usar diferentes portas para a conexão segura.

SFTP (SSH File Transfer Protocol):

  • Segurança: Funciona sobre o protocolo SSH e fornece uma maneira segura de transferir arquivos.
  • Porta: Normalmente usa a porta 22, a mesma do SSH.

TFTP (Trivial File Transfer Protocol):

  • Simplicidade: Um protocolo muito mais simples que o FTP, usado para transferências básicas de arquivos.
  • Segurança: Não possui mecanismos de segurança integrados e opera na porta 69.

FTP over TLS/SSL (FTPS):

  • Simplicidade: Um protocolo muito mais simples que o FTP, usado para transferências básicas de arquivos.
  • Segurança: Não possui mecanismos de segurança integrados e opera na porta 69.
