O que é FTP?

FTP (File Transfer Protocol) é um protocolo de rede padrão usado para a transferência de arquivos entre um cliente e um servidor em uma rede TCP/IP, como a internet. Ele permite que arquivos sejam enviados e recebidos de forma eficiente e é amplamente utilizado para gerenciar e compartilhar arquivos.

Para que serve?

FTP é utilizado para:

  • Transferência de Arquivos: Enviar ou receber arquivos entre computadores ou servidores através da internet ou de uma rede local.
  • Gerenciamento de Arquivos: Permite o gerenciamento de arquivos e diretórios em um servidor remoto, incluindo operações como criar, deletar e renomear arquivos e pastas.
  • Backup e Recuperação: Facilita a realização de backups e a recuperação de dados ao transferir arquivos entre sistemas.
  • EPublicação de Websites: É comum usar FTP para fazer upload de arquivos de um site para um servidor web.

Como funciona?

FTP utiliza duas conexões distintas para transferir dados: uma para comandos e outra para a transferência real de arquivos.

Conexão de Controle:

  • Porta: Normalmente utiliza a porta 21.
  • Função: Envia comandos e respostas entre o cliente e o servidor.

Conexão de Dados:

  • Porta: Utiliza uma porta separada, que pode ser a porta 20 no modo ativo ou uma porta dinâmica no modo passivo.
  • Função: Transferência real dos dados dos arquivos.

Modos de Conexão

Modo Ativo:

  • O cliente se conecta ao servidor na porta 21 para enviar comandos.
  • O servidor então abre uma conexão de dados na porta 20 para enviar ou receber arquivos.

Modo Passivo:

  • O cliente se conecta ao servidor na porta 21 para enviar comandos.
  • O servidor abre uma porta aleatória para a transferência de dados, e o cliente se conecta a essa porta.

Comandos FTP Comuns

  • USER e PASS: Autenticação do usuário.
  • LIST ou NLST: Listar arquivos e diretórios no servidor.
  • RETR: Baixar um arquivo do servidor.
  • STOR: Enviar um arquivo para o servidor.
  • DELE: Deletar um arquivo no servidor.
  • MKD e RMD: Criar e remover diretórios, respectivamente.
  • QUIT: Encerrar a conexão FTP.

Como foi feito?

Desenvolvimento Inicial:

  • O FTP foi desenvolvido na década de 1970 como um dos primeiros protocolos para a transferência de arquivos entre sistemas em rede.
  • A primeira especificação foi publicada em 1971, e a versão mais recente do protocolo foi formalizada em 1985.

Evolução:

  • O FTP foi inicialmente definido para ser simples e eficaz em redes locais. Com o crescimento da internet, as implementações do FTP foram adaptadas para lidar com maior segurança e eficiência.
  • Novas versões e extensões foram desenvolvidas para melhorar a funcionalidade, como FTPS (FTP Secure) e SFTP (SSH File Transfer Protocol), que adicionam segurança adicional ao processo de transferência.

Variações e Extensões

FTPS (FTP Secure):

  • Segurança: Adiciona uma camada de segurança usando SSL/TLS para criptografar a comunicação entre o cliente e o servidor FTP.
  • Portas: Utiliza as mesmas portas do FTP padrão, mas pode ser configurado para usar diferentes portas para a conexão segura.

SFTP (SSH File Transfer Protocol):

  • Segurança: Funciona sobre o protocolo SSH e fornece uma maneira segura de transferir arquivos.
  • Porta: Normalmente usa a porta 22, a mesma do SSH.

TFTP (Trivial File Transfer Protocol):

  • Simplicidade: Um protocolo muito mais simples que o FTP, usado para transferências básicas de arquivos.
  • Segurança: Não possui mecanismos de segurança integrados e opera na porta 69.

FTP over TLS/SSL (FTPS):

  • Simplicidade: Um protocolo muito mais simples que o FTP, usado para transferências básicas de arquivos.
  • Segurança: Não possui mecanismos de segurança integrados e opera na porta 69.
